## 1. 关于“大模型”，你应该知道的

### 1.1 从 AI 到大模型的演变

**人工智能（AI）**：让机器具备学习、推理、自我修正、感知、语言处理等能力的学科

* 交叉学科：计算机科学、数学、心理学等
* 多个子领域：相互关联并共同推进能力边界

### 1.2 大模型概念与关键节点

**基础模型（Foundational Models）**（2021）：

* 参数规模大（常见十亿级以上）
* 大规模数据预训练
* 具备多任务迁移能力（适配多应用场景）

**ChatGPT（2022.11）**：

* 强自然语言理解与生成
* 支持写作、邮件、脚本、翻译、代码等
* 产业影响：推动全球大模型应用浪潮

**国内模型集中发布（2023.03 起）**：

* 文心一言、通义千问内测、商量、360 智脑、盘古、讯飞认知、天工、言犀、混元等

**通义千问开源节点（2023.08 起）**：

* 多规格（7B、72B 等）
* 当前内容提到已升级至 Qwen3，能力覆盖语言、多模态、工具使用、Agent 交互等
* 训练范式：大规模预训练 + 高质量微调，贴近人类偏好

### 1.3 使用方式概览（阿里云百炼）

* 路径：百炼 → 文本模型 → 文本对话 → 更多模型 → 选择 Qwen-Plus-Latest → 立即体验
* 支持知识库检索能力以提升专业问答准确率

### 1.4 训练三阶段框架

1. **预训练（Pre-training）**：学习通用知识与统计规律
2. **监督微调（SFT）**：用标注数据对齐指令理解与任务表现
3. **对齐偏好（如 RLHF）**：对齐人类偏好与安全边界，提高可用性与稳定性

### 1.5 大模型四个典型特点

* **规模大**：参数量从数亿到数千亿，表达能力强
* **适应性强**：微调、少样本学习，迁移到多任务
* **预训练数据广**：多语种、多模态，学习通用表征
* **资源需求高**：训练与推理需要高算力、存储与能耗

## 2. 大模型是如何工作的

### 2.0 使用层面核心

* 用户输入的自然语言即 **提示词（Prompt）**
* 提示词越清晰，输出越贴近期望

### 2.1 分词化（Tokenization）与词表映射

**分词化**：把文本切分为 token

* 词粒度：适合英文等
* 字符粒度：中文可直接按字
* 子词粒度：适合新词、专有名词、网络用语，靠词根词缀组合增强覆盖
  **词表映射**：token → token id（序列化，供模型计算）

### 2.2 生成文本的自回归过程

* 机制：根据已有 token 序列预测下一个 token
* 流程：预测 → 追加到序列 → 再预测下一个
* 终止：输出结束标记（如 EOS）或达到长度阈值
* 直观理解：看似在回答问题，本质在延续提示文本的最可能续写

## 3. 大模型的应用

### 3.1 阿里云通义体系与百炼平台

**通义系列**：覆盖多领域与大量服务场景

* 基础模型：通义千问（语言与多模态理解）、通义万相（图像与视频生成）
* 应用模型：基于基础模型能力叠加行业知识、工具链、插件、流程编排等
  **百炼平台**：面向企业开发者、个人开发者、ISV
* 模型调用
* 模型训练与开发
* 应用构建全链路套件
* 插件与集成能力
* 企业专属数据与 API 融合

### 3.2 通义千问示例

* 查天气：需在模型配置开启搜索能力
* 图表解读：可用 VL 模型，先上传图片后提问
* 官网场景：会按需要自动搜索相关知识辅助回答

### 3.3 典型应用场景举例

* **通义灵码**：代码生成、解释、优化、单测生成
* **通义听悟**：语音转文字、会议纪要、字幕、内容分析
* **通义法睿**：合同审查、法律咨询、文书起草、法条定位
* **通义晓蜜**：企业客服对话，多轮问答，文档问答，坐席答案推荐并可定位出处段落
  延伸：AI 阅读助手、金融助手、角色对话智能体等

## 4. 课后小测验结论

**正确选项：B、D**

* B：多模态模型可处理文本、图像等多种数据形态
* D：企业专有知识库可在基座模型上用微调等方式定制
  错误点提示：
* 文本生成存在逐步预测 token 的过程
* 训练数据规模与质量对效果影响很大

## 5. “用好大模型”的进阶路线

* **开始使用**：遇到问题先判断能否交给模型
* **不断改进**：

  * 提示词迭代提升质量
  * 用检索增强（RAG）降低编造风险
  * 用微调稳定风格与格式、强化领域能力
